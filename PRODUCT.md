# Product

## Register

product

## Users

The primary user is a desktop writer building long-form web novels with AI assistance and WPS document output. They are usually in a focused planning or drafting session, comparing worldbuilding, character memory, plot structure, chapter goals, prompt output, and AI feedback side by side.

## Product Purpose

Novel Prompt Guide is a writing cockpit for turning a rough novel idea into a structured story bible, relationship map, chapter plan, prompts, and WPS-ready writing material. Success means the user can start from an initial idea, make clear decisions, route small AI requests to the right model or agent, and keep token cost under control without losing track of the book.

## Brand Personality

Calm, precise, writerly. The interface should feel like a serious writing desk with technical power underneath: clear enough for daily use, dense enough for complex planning, and restrained enough that the user can stay inside the story instead of fighting the UI.

## Anti-references

Do not make this feel like a video production tool, a character voice studio, a generic SaaS landing page, a neon dashboard, or a mobile-first consumer app. Avoid oversized budget sections, decorative cyberpunk effects, over-rounded cards, repeated identical card grids, unclear modal stacks, hidden API calls, and UI that makes the next writing action hard to find.

## Design Principles

1. Keep writing decisions in the foreground: every panel should answer what to write, what to check, or what to ask AI next.
2. Make AI calls explicit: no automatic external API work unless the user clicks a clear action.
3. Treat windows as work surfaces: every feature window should be movable, resizable, and useful on its own.
4. Show relationships and process spatially: character lines, plot causes, and stage flow should be visible without becoming decoration.
5. Prefer calm density: desktop users need many controls at once, but spacing, labels, and hierarchy must keep the surface readable.

## Accessibility & Inclusion

Target practical WCAG AA contrast for text and controls. Use clear focus states, predictable button labels, readable Chinese typography, and reduced-motion alternatives where motion is present. The default experience is desktop-first; mobile support can be functional but should not drive the layout.
