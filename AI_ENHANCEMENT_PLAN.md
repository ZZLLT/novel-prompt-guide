### 阶段3：AI智能建议系统（3小时）

#### 3.1 上下文感知建议
**功能：** AI根据当前创作状态主动提供建议

**实现位置：**
- AI助手面板底部
- 显示3-5条智能建议
- 点击即可执行

**建议类型：**
```
📝 创作建议
- "你的主角还没有明确的动机，要添加吗？"
- "场景3缺少冲突点，需要我帮你设计吗？"
- "剧情线A进展太快，要添加铺垫吗？"

🔍 质量检查
- "角色B在场景5的行为与性格不符"
- "时间线存在矛盾：场景3应该在场景5之前"
- "对话过于平淡，需要润色吗？"

💡 创意激发
- "可以添加一个转折点：让反派发现主角的秘密"
- "建议添加一条感情线，让故事更丰满"
- "这里可以埋一个伏笔，后续揭示"
```

**API设计：**
```typescript
GET /api/ai/suggestions?context={...}

Response:
{
  suggestions: [
    {
      type: "warning" | "tip" | "idea",
      title: "角色动机不明确",
      description: "主角林风的行为缺乏动机支撑",
      action: {
        command: "帮主角添加一个清晰的动机",
        entity: "character-123"
      }
    }
  ]
}
```

### 阶段4：批量生成功能（2小时）

#### 4.1 批量生成角色
**功能：** 一次生成多个相关角色

**用例：**
```
输入："生成5个角色：主角、师父、师姐、反派、配角"
输出：一次性返回5个完整角色
```

**API设计：**
```typescript
POST /api/ai/generate/batch/characters
{
  prompt: string,
  count: number,  // 生成数量
  relationships: boolean  // 是否自动建立关系
}
```

#### 4.2 批量生成场景
**功能：** 根据大纲批量生成章节

**用例：**
```
输入："根据这个大纲生成10个场景"
输出：完整的10个场景，自动关联角色和剧情线
```

### 阶段5：AI自动检测和修复（2.5小时）

#### 5.1 自动检测问题
**功能：** 定期扫描，自动发现问题

**检测维度：**
```
1. 角色一致性
   - 性格前后矛盾
   - 能力突然改变
   - 关系不合理

2. 剧情逻辑
   - 时间线冲突
   - 因果关系断裂
   - 剧情漏洞

3. 场景质量
   - 对话不自然
   - 描写过于简单
   - 缺少冲突点

4. 整体结构
   - 节奏问题
   - 伏笔未回收
   - 主线不清晰
```

#### 5.2 一键修复
**功能：** AI自动修复检测到的问题

**工作流：**
1. 用户点击"自动检测"
2. AI扫描所有内容
3. 列出问题清单
4. 用户选择要修复的问题
5. AI批量修复

#### 6.1 续写场景

**功能：** 专门优化对话部分

```typescript
POST /api/ai/optimize/dialogue
{
```
